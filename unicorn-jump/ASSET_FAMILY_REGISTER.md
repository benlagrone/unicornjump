# Asset Family Register

## Purpose

This is the explicit all-families register.

Use it when the grouped matrix is too abstract.

Rule:

- every active family must be listed
- every listed family must have a current tier
- every listed family must have a target tier
- every listed family must have a named next pass

No grouped production blind spots are allowed.

## Tier Key

- `Good`
- `Better`
- `Best`
